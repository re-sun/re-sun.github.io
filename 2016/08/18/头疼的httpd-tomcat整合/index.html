头疼的httpd+tomcat整合

By Sun

2016-08-18 更新日期:2016-08-18

今天折腾了一早上，刚才差点给跪了，因为在远程服务器上我不知道原先已经安装了httpd，后来我又用yum安装了一遍，然后修改各种配置，再重启。。。

重启的时候提示我，端口已经被占用。。

Starting httpd: (98)Address already in use: make_sock: could not bind to address 0.0.0.0:80
no listening sockets available, shutting down

好吧，于是我使用命令查看到底是谁占用了80端口

netstat -lnp | grep 80

结果是这样。。

tcp 0 0 0.0.0.0:8080 0.0.0.0: LISTEN 20409/java
tcp 0 0 0.0.0.0:80 0.0.0.0: LISTEN 1321/httpd
tcp 0 0 0.0.0.0:8085 0.0.0.0: LISTEN 11688/nginx
tcp 0 0 127.0.0.1:8005 0.0.0.0: LISTEN 20409/java
tcp 0 0 0.0.0.0:8009 0.0.0.0:* LISTEN 20409/java

恩。。httpd。。。

然后毫不犹豫的杀掉所有httpd进程

pkill httpd

接着重启

service httpd restart

卧槽。。

奔溃了。。

情报君的网站打不开了。。

提示找不到首页的php。。。

要跪了。。

后来想，应该是两个httpd的问题，我重启的并不是之前所用的http

然后找到之前所用的httpd路径，找到bin目录，执行命令

./httpd

情报君网站终于恢复了。。

一大早搞的我都没心情吃饭了。。
